Build Order

Recommended Build Sequence

  1. Schema first: update Prisma schema and create the migration before UI or API code depends on it.
  2. Server second: add backend routes, guards, jobs, and data validation.
  3. Client third: wire pages, tabs, and dashboard views only after API contracts are stable.
  4. Local validation fourth: verify migration deploy, build output, and critical workflows.
  5. Image build fifth: build and push the new Docker image.
  6. Azure rollout last: update ACA to the exact image digest, then verify production.
